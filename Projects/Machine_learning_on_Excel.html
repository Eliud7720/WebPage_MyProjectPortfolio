Exploratory Data Analysis and Preprocessing of EGFR, MAPK1, SRC, and STAT3 Inhibitors Reported in the ChEMBL Compound Library

About this project

Dementia, primarily Alzheimer’s Disease (AD), is one of the leading causes of disability and mortality in older adults, affecting 1.3 million people in Mexico. Type 2 diabetes (T2D) increases the risk of developing AD by 1.5 to 2 times and is associated with cerebral insulin resistance, inflammation, and oxidative stress. Antidiabetic drugs have shown cognitive improvements in patients with AD or Mild Cognitive Impairment. To date, no specific inhibitors targeting proteins involved in the AD-T2D interaction have been identified. Advances in artificial intelligence and machine learning provide tools to discover new inhibitors with high precision and efficiency, optimizing drug development. Therefore, employing machine learning models could enable the identification of potential inhibitors for the AD-T2D association with high accuracy, improve their selection, and reduce the costs associated with drug development. A study identified a total of 10 proteins linking AD with T2D, with EGFR, MAPK1, SRC, and STAT3 being the most prominent. Given the large number of assays reporting IC50 values, machine learning models can be trained to predict IC50 activity for each protein.

Objective

Analyze and preprocess data on EGFR, MAPK1, SRC, and STAT3 inhibitors reported in the ChEMBL compound library to obtain clean datasets and identify compounds shared among all four proteins.

Development

Data collection

Experimental data on inhibition was collected from the ChEMBL web chemolibrary for the proteins EGFR, MAPK1, SRC, and STAT3, which have IDs CHEMBL203, CHEMBL4040, CHEMBL267, and CHEMBL4026 in CSV format. Each file was loaded as a pandas dataframe in Python. The compound data for each file consisted of 47 columns describing each assay of a specific compound against the protein, with columns including information such as the compound name, SMILES (a way to represent chemical compounds in cheminformatics), and IC50 assay results. Only the following columns were selected: "Molecule ChEMBL ID", "Molecule Name", "Smiles", "Standard Type", "Standard Relation", "Standard Value", "Standard Units", "Assay Type", "Molecular Weight", "BAO Label", "Action Type".

Data Preprocessing

For the data of each protein, the amount of missing data in each column was analyzed using the pandas .isna().sum() method. Results like the following were obtained:

In the previous table, it can be seen that there are missing values in the columns Molecule Name, Smiles, Standard Relation, Standard Value, Standard Units, and Molecular Weight. Rows with missing values in Standard Relation, Standard Value, Standard Units, or SMILES were removed using the dropna() method. However, rows with missing values in Molecule Name were retained, since in many cases molecules do not yet have an assigned name, and rows with missing values in Molecular Weight were kept because if the molecule's SMILES is present, it can be used to calculate the molecular weight.

The data in the Standard Units column was analyzed to check whether the assay data had consistent units and, if not, to perform the necessary conversions when possible. This analysis was carried out using the value_counts() method. The following table shows an example of the units determined with value_counts().

Standard Units Count
nM 15805
ug.mL-1 86
/uM 6

As shown in the previous table, the majority unit type in the assays is nM; however, there are 86 assays with units ug.mL-1 and 6 with units /uM. The necessary unit conversions were carried out using masks and the corresponding equations with pandas.

Subsequently, duplicates in the Molecule ChEMBL ID and Molecule Name columns were removed in order to retain only a single random assay per molecule.

Additionally, only assays with exact values or results, recorded in the "Standard Relation" column as "=", were retained, and assays with censored results, i.e., values marked as ">" or "<", were discarded because working with censored data is not recommended in machine learning models.

Regarding the Assay Type column, only rows corresponding to protein binding assays, denoted as "B", were retained. In the BAO Label column, only rows with assays labeled as "single protein format" were kept.

Conclusions

  • The preprocessing of ChEMBL data for EGFR, MAPK1, SRC, and STAT3 reduced the number of compounds significantly, ensuring cleaner datasets for machine learning models.
  • Missing data was carefully handled: rows with missing values in key experimental results were removed, while columns like Molecule Name and Molecular Weight were preserved when possible.
  • Unit inconsistencies in Standard Units were standardized, ensuring assay results are comparable across proteins.
  • Only reliable assays were retained by filtering for exact values (“=”) and excluding censored data (">", "<").
  • Duplicates were removed to avoid bias, and only protein binding assays and single protein formats were conserved.
  • A binary classification column (Active vs. Inactive) was created using an IC50 threshold of 1000 nM, enabling supervised learning approaches.
  • Most datasets showed class imbalances below 40%, which is manageable for machine learning algorithms with proper handling.
  • Shared compound analysis revealed overlaps among proteins (e.g., 392 compounds shared between SRC and EGFR), highlighting potential multitarget inhibitors.
  • These findings demonstrate the feasibility of using machine learning to predict IC50 values and discover new compounds that could inhibit multiple proteins related to Alzheimer’s Disease and Type 2 Diabetes.
